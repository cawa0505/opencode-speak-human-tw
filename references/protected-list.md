# 保護清單與誤殺防護

去 AI 味最大的風險不是改不乾淨，是把事實一起改掉。動筆前先把這些內容圈起來，改寫全程原封不動；改完回頭逐項核對。

## 五類保護對象

### 1. 價格與數字

定價、優惠碼、折扣百分比、名額、日期、時間、統計數據、成效指標。

- 「早鳥價 4,800 元、折扣碼 EARLY500、只到 3/31」逐字保留
- 改寫不能讓數字的精確度漂移（「約 42%」不能變「超過四成」，反之亦然）
- **絕不為了「更具體」補上原文沒有的數字**：具體化靠作者提供事實，不靠編

### 2. 專有名詞

課程名、品牌名、產品名、活動名、專欄名。

- 不因「刻意換詞循環」規則被同義替換：《超級個體工作術》第二次出現還是《超級個體工作術》，不會變成「這門課程」（口語簡稱除外，需與作者慣用一致）
- 中國平台與產品的專名照原文（小紅書、公眾號），不套台灣用語替換表

### 3. 網址與連結文字

CTA 連結、報名連結、社群連結、文章引用連結。

- 連結本體與錨點文字保留
- 例外：連結上的 AI 工具參數（`utm_source=chatgpt.com`、`utm_source=openai`、`referrer=grok.com`）要清掉，這屬於工具痕跡不屬於連結內容；作者自己設定的 UTM 行銷參數（`utm_source=newsletter`）保留

### 4. 真實姓名與引號內原話

見證、訪談、學員故事、對話紀錄。

- 真名不可改成「這位學員」「某位夥伴」這類模糊代稱
- 引號內的原話一字不改，包括裡面的中國用語、半形標點、AI 味句型：引用的價值就是原汁原味
- 原話有明顯錯字時標註提醒作者，不代改

### 5. 承諾類文字

退費政策、保固條款、免責聲明、服務範圍聲明。

- 只能調前後銜接的語氣，不能調意思
- 「14 天內」「全額」「不需要任何理由」這類量詞與條件一字不漂
- 金流說明的制式語言（「依發卡銀行作業為準」）是準確性需求，不是罐頭腔

## 容易誤殺的情況

規則對了，判斷錯了，一樣改壞。這些情況先放行：

| 看起來像 | 其實是 | 判斷方法 |
| :-- | :-- | :-- |
| 排比三段式 | 承載真實對比的修辭 | 排比底下有沒有具體事實接著？有就放行 |
| 無源引用 | 有來源的數據 | 「根據我們後台的數據」有明確出處，放行 |
| 「不是 A 而是 B」 | 全文唯一一次的核心轉折 | 規則是整篇最多一次，不是零容忍 |
| 客服罐頭腔 | 金流／條款的必要制式語 | 涉及錢、法律、責任的制式語保留 |
| 重複句式 | 長文的節奏設計 | 重複有沒有在推進敘事？有就是設計 |
| AI 味詞彙 | 被討論的對象 | 「我戒掉了『賦能』這個詞」是提及不是使用 |
| 正式官腔 | 公告場景的合宜語域 | 停機公告本來就該正式 |
| 空泛的「優化」 | 掛著具體動作與指標 | 後面接得出「改了什麼、變化多少」就放行 |
| 口語碎片、不完整句 | 人味本體 | 這是要保護的方向，一個字都不動 |

拿不準的通用判斷法：**這句話刪掉或改掉之後，讀者少知道了什麼？** 少知道了任何事實、立場或情緒，就不能刪；什麼都沒少，才是空話。

## 回讀核對程序

改寫完成後，依序執行：

1. 把改寫前圈出的保護對象列成清單（價格 N 項、名詞 N 項、連結 N 項、真名與原話 N 項、承諾 N 項）
2. 在改寫版逐項搜尋確認存在且逐字一致
3. 確認沒有新增原文沒有的事實、數字、來源
4. 確認作者立場沒有翻轉或稀釋（「我反對」不能變「我持保留態度」）
5. 任何一項失敗：回去修正，不交稿
